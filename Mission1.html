1. Contexte

L’objectif de cette mission est de prendre en main le sujet et les données fournies par l’entreprise Kdou. Pour cela, un classeur Excel a été mis à notre disposition, avec une feuille par table de base de Données. Ce support nous a permis de répondre à des questions spécifiques de l’entreprise. La base de données est composée de 6 tables : Fournisseur, Produit, Catégorie, Clientèle, Commande, DétailCommande.

2. Bilan

La mission est divisé en 3 parties. Dans une première partie, nous effectuons une analyse des clés primaires, puis secondaires et enfin nous finissons par la réalisation de requêtes sur le fichier EXCEL. Analyse des clés primaires : Les données sont structurées en plusieurs tables avec des clés primaires pour chaque table (comme N°Fournisseur pour Fournisseur ou Code Client pour Clientèle) permettant d’identifier chaque entrée de façon unique. Les clés étrangères relient les tables entre elles : par exemple, Produit contient le N°Fournisseur et le Code Catégorie pour lier chaque produit à son fournisseur et sa catégorie. La table Commande utilise le Code Client pour associer une commande à un client, tandis que DetailCommande relie chaque détail de commande à un produit et à une commande grâce aux références Ref produit et N°Commande.

2.2 Exemple requête

Afin d'obtenir une information précise, nous avons apppliqué des filtres, pour réduire les tables à la réponse souhaité. Voici quelques exemples de requêtes:

REQUETE : Combien y a-t-il de clients de région parisienne ? Réponse : Il y a 3 clients de région parisienne
Code Client Société Contact Ville Pays
ALYAD Chez Bouxou Alyssa Duranta Paris France
FAMIA ChefFamilia Aria Cruz Paris France
FISSA Une petite faim Diego Roel Paris France
REQUETE: Combien y a-t-il de pays dans l’UE autre que la France ? Réponse: il y en a 7.

Cet exemple illustre bien les limites de la méthode de filtrage. Tout d'abord, EXCEL ne permet pas de compter directement les lignes, ce qui nous oblige à faire un décompte manuel des pays, alors que le fichier est censé faciliter la lecture des données. De plus, pour sélectionner les pays de l'UE, chaque pays a dû être choisi individuellement, une manipulation qui augmente le risque d'erreurs par rapport à une requête "automatisée" dans une base de données.

3. Perspective d'amélioration

Nous avons remarqué certains problèmes sur les données de l’entreprise qui pourraient ralentir son développement, impactant l’efficacité et la rentabilité de KDou : Par exemple : Sao polo et São polo apparaissent comme deux entrées distinctes dans la table des fournisseurs, bien qu’il s’agisse de la même ville. Cela résulte en une double entrée pour un même lieu. Pour y remédier on pourrait normaliser les données en appliquant des règles d’écriture uniformes et standard. De plus on retrouve des incohérences dans les intitulés : Royaume-Uni est écrit deux fois, une fois avec un tiret et une fois sans, et les titres de certains postes (ex. : représentante vs. représentant) présentent des variations d’orthographe. Pour réagir face à ce problème on pourrait mettre en place un logiciel permettant de détecter et corriger, les doublons et fautes d’orthographes. Certains codes clients sont référencés sans que le client n’ait passé de commande, comme FRANCK, ce qui suggère des erreurs ou des données manquantes. Pour y remédier, on pourrait faire en sorte de vérifier les données avant son importation dans la base.

4. Conclusion

L’étude de données révèle une organisation assez satisfaisante. La plupart des requetes demandées par l'entreprise KDOu ont pu être réalisé avec succès. Cependant, certains éléments de la base de données, pourraient être plus optimisés afin d'assurer une meilleure qualité de l'information et donc une meilleure performance de l’entreprise. En effet, le filtrage des colonnes est efficace, mais reste limité. EXCEL n'est peut etre pas le meilleur outil pour ce type de requête. Nous suggerons plutot l'utilisation d'ACCESS, qui est un outil similaire mais plus précis encore, avec l'usage de langage tel que SQL et QBE.
